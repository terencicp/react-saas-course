If you have shipped something on the web before, are returning to it after time away, and want to operate at senior depth on a modern SaaS stack — this is for you. By the end of the course you can ship a real production SaaS feature on the 2026 stack, and the result holds up to a senior code review.

## Who this course is for

You've shipped something — a side project, a freelance build, a job that ended a while back. You remember writing JavaScript, you've used a terminal, you've deployed a thing somewhere. You don't remember closure capture rules; that's fine. You want to be the engineer who decides what an API contract should look like before writing a route, not the one who copies a route from a tutorial.

This is not a first-week-of-coding course, and it is not a "ten years of backend, dabbling in frontend" course. Both bounce off the assumptions in different directions.

## Should you start here?

A short, private self-check. No scoring — the recommendation is just where your time spends best right now.

## The two pillars

Every course makes cuts. This course makes them through two filters.

**Systems design and senior mindset over syntax.** Code is a commodity in 2026. The contribution that compounds is what gets decided before code is written — planning, data modeling, API contract design — and how the code is shaped so it stays changeable. Every lesson foregrounds the decision, then shows the code.

**The minimum viable 2026 stack.** The smallest set of technologies a SaaS startup would actually ship with in 2026, and nothing else. Every tool is either a platform default (the choice when nothing tells you otherwise) or earns past a named trigger. Anything that doesn't pass one of those two tests is not in the course.

The date matters. The course pins May 2026 because the right answer drifts. When a tool wins in 2026, the senior reason is named — so you can spot the day it stops winning.

## How to read this course

Two chapter shapes, alternating.

**Teaching chapters** install concepts and end with a short quiz — recall, not a graded gate. Wrong answers point at the lesson that taught the concept.

**Project chapters** are the practice. You ship a real surface — a working feature on the 2026 stack, not a toy exercise — in 1.5 to 3 hours, on the same codebase you've been carrying forward across the course. By the time you reach the later units, the codebase contains everything from Unit 1 onward. No rebuilding from scratch.
