# Toolchain constraints

Configuration knobs the toolchain forces, breaks, or mishandles. Plans locking these must follow the listed resolution. Append entries when reviewer/corrector finds a new framework conflict.

## Next.js 16

- **`tsconfig.json: "jsx"`** — `next build` rewrites `"preserve"` → `"react-jsx"` ("mandatory change"). Pin `"jsx": "react-jsx"` in plans.
- **`next-env.d.ts`** — auto-regenerated with double-quoted imports, conflicts with Biome `quoteStyle: 'single'`. Exclude from Biome: `"files": { "includes": ["**", "!next-env.d.ts"] }`.
- **Multi-lockfile warning** — emitted when a parent dir has its own lockfile (monorepo case). Silence with `turbopack: { root: __dirname }` in `next.config.ts`.
- **`tsconfig.json` auto-injection** — each `next build` appends `".next/dev/types/**/*.ts"` to `include` and reformats arrays multiline. It also suggests `"allowJs": true`, but *only when the option is absent* — an explicit `"allowJs": false` survives. Of all the changes Next prints, only `jsx` / `module` / `moduleResolution` / `esModuleInterop` / `isolatedModules` / `resolveJsonModule` are labeled "mandatory"; the include addition and `allowJs` are "suggested." Bake the include addition and the `jsx: "react-jsx"` pin into plans' locked `tsconfig.json` so slices running `next build` don't re-introduce drift.
- **Cache Components + slot `default.tsx` re-export** — under `cacheComponents: true`, parallel-route slot `default.tsx` files re-exporting their `page.tsx` (`export { default } from './page'`) propagate the page's dynamic data reads into sibling routes' prerender. The slot-scoped `@<slot>/loading.tsx` only acts as a Suspense boundary for the slot's *own* `page.tsx`, not when composed into a sibling route. Result: build fails with "Uncached data was accessed outside of `<Suspense>`" on every sibling route. Resolution: add segment-level `loading.tsx` at the parallel-routes parent (`app/<segment>/loading.tsx`), or move dynamic reads behind `'use cache'`. Hand-written `<Suspense>` is last resort; segment-level `loading.tsx` is the cleanest seam.
- **Parallel routes implicit `children` slot needs `default.tsx`** — when a layout declares named slots, the implicit `children` slot also needs its own `default.tsx` at the same level. Without it, URLs matching only the named slots render a broken page: HTTP status is **200**, the named slots render normally, but the children area shows Next's "This page could not be found" fallback (a `NEXT_HTTP_ERROR_FALLBACK;404` template inlined into the slot's DOM). Always enumerate `app/<segment>/default.tsx` alongside `@<slot>/default.tsx` files in plans.
- **`typedRoutes: true` + dynamic href strings** — template-literal `href={`/path/${id}`}` and `router.push("/path?x=y")` with built strings fail typecheck because `Route` is a branded string union. Caveat: the augmentation lives in `.next/types/link.d.ts`, but Next 16's default `next-env.d.ts` only references `routes.d.ts`. The `next` TS server plugin loads `link.d.ts` in the IDE, but **`tsc --noEmit` silently passes** unless plans also add `/// <reference path="./.next/types/link.d.ts" />` to `next-env.d.ts` (or include it manually). Resolution: cast site-by-site (`as Route` or `as Parameters<typeof router.push>[0]`), or route href construction through a typed helper. Plans flipping `typedRoutes: true` should pre-state both the reference-path wiring and the cast pattern.
- **`new Date()` in Server Components under Cache Components** — reading `new Date()` (or any IO clock source) inside a Server Component without first awaiting a dynamic source breaks `next build` under `cacheComponents: true`. Surfaces most often in inherited footers (`© {new Date().getFullYear()}`). Resolution: hardcode the literal year, move into a Client Component, or precede the read with an `await` on a dynamic source.

## Biome 2.4

- **`css.parser.tailwindDirectives`** — Tailwind v4 directives (`@theme`, `@apply`, `@custom-variant`, `@import "tailwindcss"`) emit parse errors without this flag. Always include `"css": { "parser": { "tailwindDirectives": true } }` in `biome.json`.
- **`lint/performance/noImgElement`** — top-level performance rule that warns on raw `<img>` (no `next` domain exists in Biome 2.4; framework filtering happens via `--only=next`, not a domain block). Plans mandating raw `<img>` (theme-aware image swaps, plain logos) must override: `"lint": { "rules": { "performance": { "noImgElement": "off" } } }`.
- **`files.includes` folder-glob normalization** — `useBiomeIgnoreFolder` rejects `"!folder/**"` and `biome check --write` auto-strips to `"!folder"`. Plans pinning the `/**` suffix on ignore entries fail `biome ci` until rewritten. Write folder ignores without trailing glob: `["!.next", "!node_modules"]`, not `["!.next/**", "!node_modules/**"]`.
- **`<ul role="list">` Safari VoiceOver workaround** — the standard Safari fix for `list-style: none` stripping AT list semantics trips `lint/a11y/noRedundantRoles` (only — `useSemanticElements` does not target this case in Biome 2.4). Resolution: suppress per-element with a `biome-ignore` comment, or disable the rule globally in `biome.json` when the project commits to the workaround. Architects mandating the role must pre-declare the chosen reconciliation.

## Shadcn UI

- **Primitive formatting** — `pnpm dlx shadcn add` writes primitives with double quotes and `import * as React` (not `import type`), triggering Biome warnings on first lint. Accept on first commit; let `pnpm check --write` normalize.
- **Radix umbrella package** — shadcn's `new-york-v4` style depends on the `radix-ui` umbrella package rather than per-primitive `@radix-ui/react-*`. (Older styles like `default` and `new-york` still use per-primitive packages.) For v4-style installs, pin `radix-ui@^1.4.3` instead of enumerating per-primitive deps.
- **`Card` has no `asChild` slot** — shadcn's `Card` renders a `<div>` with no `asChild` prop, so it can't be retargeted to `<article>` or other semantic roots. Plans needing a semantic root (e.g. `FeatureCard` typed as `ComponentProps<'article'>`) must inline `Card`'s container classes onto the chosen element and nest `CardHeader` / `CardTitle` / `CardDescription` inside, rather than wrapping `Card`. Compose-by-hand, don't double-wrap.
- **Radix primitives ship `'use client'`** — `dialog`, `sheet`, `separator`, and other Radix-wrapping primitives carry the directive. Any verification check counting `'use client'` files must include the installed `components/ui/*` set, not just authored islands.

## Postgres 18 (Docker)

- **Volume mount path** — Postgres 18 official image stores data under `/var/lib/postgresql/18/docker` (the new major-versioned layout). Mount the parent `/var/lib/postgresql` so `pg_upgrade --link` works cleanly across major-version bumps; the historic `/var/lib/postgresql/data` is still writable but breaks the upgrade flow. Use:
  ```yaml
  volumes:
    - pgdata:/var/lib/postgresql
  ```
- **`@neondatabase/serverless` cannot reach vanilla Postgres** — the Neon driver speaks HTTP/WebSocket only, not Postgres wire protocol; against local Docker it errors before connect. For local Docker chapters, use the course-pinned `postgres` (postgres-js) driver via `drizzle-orm/postgres-js` — it speaks the wire protocol, bundles cleanly under Turbopack, and needs no `serverExternalPackages` entry. 

## Drizzle 0.45

The course is pinned to `drizzle-orm@0.45.x` + `drizzle-kit@0.31.x` until Better Auth ships `drizzle-orm@^1.0` support ([better-auth#6766](https://github.com/better-auth/better-auth/issues/6766), target 1.8+). Constraints on this line:

- **Runtime `casing` lives on the client.** `drizzle({ client, casing: 'snake_case', schema })` snake-cases all columns from one declaration. Per-column overrides via explicit names (`uuid('organization_id')`) are the escape hatch. (Removed in Drizzle 1.0 — flips to `pgTableCreator((name) => toSnakeCase(name), 'snake_case')`; the second argument is mandatory or columns stay camelCase.)
- **Migration file layout is flat.** `drizzle-kit generate` emits `drizzle/<timestamp>_<name>.sql` plus a top-level `drizzle/meta/_journal.json` and `drizzle/meta/<id>_snapshot.json`. Pass `--name <verb>_<noun>` so the suffix is intentional from the start. Renaming the timestamp prefix breaks journal lookup. (Drizzle 1.0 moves to per-migration directories `drizzle/<timestamp>_<name>/{migration.sql, snapshot.json}` with no top-level `meta/`.)
- **Relations v1 is the API.** `relations(<table>, ({ many, one }) => ({ ... }))` declared per-table; `db.query.<table>.findMany({ with: { ... } })` is the N+1-safe traversal. (Drizzle 1.0 replaces this with a single `defineRelations(schema, (r) => ({ ... }))` call; the `r.one` form defaults to `optional: true` and must be flipped to `false` on NOT NULL FKs.)
- **Zod helpers ship as a separate package.** `drizzle-zod@^0.8` peer-requires `drizzle-orm@>=0.36` and `zod@^4`. `createSelectSchema` / `createInsertSchema` / `createUpdateSchema` import from `drizzle-zod`, not `drizzle-orm/zod`. (1.0 drops the separate package and moves them under `drizzle-orm/zod`.)
- **`drizzle-seed` `with` keys are table names, not relation aliases** — refining a parent with children uses the schema's table binding (`with: { invoiceLines: [...] }`), not the relation alias (`lines:`). Junction tables hand-populated post-seed need `{ count: 0 }` in the refine to suppress the default fill. This is general across both versions.
