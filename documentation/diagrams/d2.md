# D2

> _This document distills lessons learned from diagrams already built in this course. Treat it as a kick-starter for new diagrams, not a strict guide._

`astro-d2` is wired in [astro.config.mjs](../../astro.config.mjs) with the D2.js WASM build, ELK layout, and `inline: true` (icons/themes need inline). Author D2 in fenced ` ```d2 ` blocks inside MDX — each block becomes an inline SVG. Live reference: [`src/content/docs/0 Demos/diagrams/d2-demo.mdx`](../../src/content/docs/0%20Demos/diagrams/d2-demo.mdx).

## Boilerplate

```mdx
import Figure from '../../../../components/figures/Figure.astro';

<Figure caption="Short caption (Unit X)">
```d2
direction: right
…
```
</Figure>
```

Every D2 block must be wrapped in `<Figure>` ([guidelines](./INDEX.md)).

## Prefer left-to-right

**Always start with `direction: right` unless the diagram has a strong reason to flow top-down** (e.g. a sequence-shaped state machine, or an ER schema whose tables stack naturally). Laptop viewports are wide but short, and the figure card hits its height budget long before its width budget. A top-down layout that fits a 3-row hub-and-spoke fan (e.g. inputs → server → services) commonly renders 600–800px tall; the same graph as `direction: right` lands closer to 350–450px while staying inside the card width. Switch only if a `direction: right` version is demonstrably worse to read.

## Shapes we use

- Default = rectangle. Use bare `name: "Label"`.
- **Rounded rectangle**: there is no `rounded` shape. Use `style.border-radius: 20` on a default rectangle. `shape: oval` is a *pill*, not a rounded rectangle — don't confuse the two.
- `shape: diamond` for decisions.
- `shape: page` for file icons (see App Router demo).
- `shape: sql_table` for ER schemas, with `{constraint: primary_key | foreign_key | unique}` on columns.
- `shape: circle` with explicit `width`/`height` for state-machine start nodes.

## Containers

Nest with `{}`. The container's label is the key; children lay out inside it:

```d2
outer: "Container label" {
  a
  b
  a -> b
}
other -> outer.a   // reference children with dotted path
```

For grid layouts inside a container, add `grid-columns: N` and `grid-gap: 32`.

## Edges

```d2
a -> b                                  // plain
a -> b: "label"                         // labelled
a -> b: { style: { stroke: "#aa3322"; stroke-dash: 4 } }   // styled, no label
a -> b: "label" { style.font-size: 24 } // styled + label
```

`stroke-dash: 4` is a good fit for de-emphasised edges (third-party hops, failure branches).

## Reusable styles — `classes`

```d2
classes: {
  rounded: { style: { border-radius: 20; font-size: 28 } }
  muted:   { style: { fill: "#e5e7eb"; stroke: "#6b7280" } }
}

a: "Some node" { class: rounded }
b.class: muted     // also works as a flat property
```

## Font size — bump it on dense diagrams

D2 + ELK lays out at "natural" coordinates, then the SVG is downscaled to the figure card's width. A dense `direction: right` diagram can produce a viewBox several thousand pixels wide that the card squeezes to ~640px, making default-sized labels barely legible.

Fix with a recursive glob at the top of the block:

```d2
direction: right

**.style.font-size: 28
```

The `**` glob applies the property to every shape in the diagram, including container children. `*` is top-level only — usually not what you want. Per-node `style.font-size: N` overrides the glob when you need a specific label larger or smaller.

Rule of thumb: 24–28pt when the viewBox is roughly 3× the card width; bump further for denser diagrams.

## Failure mode

A D2 parse error makes the figure silently render without an SVG — no console error, no build error from `astro-d2` in dev. If the `<Figure>` is empty after an edit, suspect a typo (most often: missing `{}`, a stray comma in a `style` block, or an unrecognised property name). Bisect by reverting the last block of changes.
