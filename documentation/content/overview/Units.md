# The course

> A one-line compressed view of the [full Table of contents](../Table%20of%20contents.md). Each line names the tech, libraries, APIs, and architectural patterns introduced in that unit's teaching chapters (project chapters excluded — they apply what the teaching chapters cover).

- **Unit 1 — Setup and Toolchain**: Node 24 LTS, pnpm, mise, TypeScript strict, Biome, Next.js 16, React 19, Tailwind v4, `@t3-oss/env-nextjs`, Zod
- **Unit 2 — JavaScript and TypeScript**: TypeScript (generics, narrowing, discriminated unions, branded types), ES modules, Promises, async/await, AbortController, Temporal, JSON
- **Unit 3 — HTTP and the Browser Platform**: HTTP semantics, fetch, DOM, event handling, Server-Sent Events, Web Storage, cookies and `Set-Cookie` attributes
- **Unit 4 — React, JSX, and Tailwind**: React 19, JSX, Tailwind v4, shadcn/ui, Radix UI, class-variance-authority, OKLCH design tokens, dark mode, container queries, responsive design. Project: a themed product surface built from shadcn/ui primitives.
- **Unit 5 — Next.js and the App Router**: App Router, Server/Client Components, Cache Components, Partial Prerendering, Suspense streaming, `proxy.ts`, `next/image`, `next/font`, `next/script`, metadata API. Project: a list-plus-detail surface using parallel routes with the modal-with-real-URL pattern.
- **Unit 6 — Postgres and Drizzle**: Postgres on Neon, Drizzle ORM, Drizzle Relations v2, Drizzle Kit migrations, UUIDv7, indexes, transactions, `$inferSelect`/`$inferInsert`. Project: the org-scoped invoicing data layer (schema, relations, migrations, seeds).
- **Unit 7 — Forms, Validation, and Server Actions**: Zod 4, Server Actions, `useActionState`, native HTML forms, FormData, drizzle-zod, route handlers, `authedAction` wrapper, Result types. Project: a full CRUD surface implemented via Server Actions on the invoicing data layer.
- **Unit 8 — Transactional Email**: Resend, React Email, sender identity (SPF/DKIM/DMARC), deliverability. Project: the welcome email send path with a React Email template delivered through Resend.
- **Unit 9 — Authentication with Better Auth**: Better Auth, Drizzle adapter, email+password, session cookies, email verification, password reset, CSRF. Project: an end-to-end email+password auth flow with email verification.
- **Unit 10 — Organizations and RBAC**: Better Auth organizations plugin, RBAC, invitations, seat-handoff lifecycle, audit log, multi-tenancy. Project: organizations, roles, and invitations wired end-to-end on the app.
- **Unit 11 — Lists, URL State, and Soft Delete**: URL state with `searchParams` + nuqs, soft delete / archive / restore, lifecycle columns, optimistic concurrency. Project: the production list view with URL-driven filters, sorting, pagination, and archive/restore.
- **Unit 12 — Webhooks and Stripe Billing**: Stripe (Checkout, Customer Portal, subscriptions, signing), webhook ingestion pattern, `processed_events` idempotency, plan entitlements. Project: a Stripe webhook pipeline that projects subscription events into a plan-entitlements row.
- **Unit 13 — Background Work and Object Storage**: Vercel `after()`, Vercel Cron, Trigger.dev v4 (queues, schedules, waitpoints, idempotency keys), Cloudflare R2, presigned URLs. Projects: a durable CSV export job on Trigger.dev, and a presigned-URL upload flow to R2.
- **Unit 14 — Notifications**: Notification dispatcher pattern, multi-channel send (email + inbox), notification preferences, dedup window. Project: a notification dispatcher fanning three events across email and in-app inbox channels.
- **Unit 15 — Cache and Rate Limiting**: Tag-driven cache invalidation (`updateTag` / `revalidateTag` / `revalidatePath`), Upstash Redis, `@upstash/ratelimit`, Vercel WAF, dual-keyed limiters. Projects: tag-driven cache invalidation on the invoices list, and Upstash rate limits on the auth surface.
- **Unit 16 — TanStack Query and Zustand**: TanStack Query v5 (`useQuery`, `useMutation`, `useInfiniteQuery`), Zustand v5 (`createStore`, slices), optimistic updates with rollback, polling. Projects: a polling, infinite-scrolling optimistic comment thread with TanStack Query, and a routed multi-step customer wizard with Zustand.
- **Unit 17 — Errors and Security**: Fail-closed error discipline, CSP with nonces, security headers, rate-limit coverage, audit log schema, GDPR retention, consent gating, pnpm 11+ supply-chain defaults. Project: a pre-launch audit pass that hardens error handling, headers, and the security baseline across the app.
- **Unit 18 — Time and Internationalization**: Temporal API, `timestamptz`/`date` codecs, IANA timezones, DST scheduling, `Intl.*` formatters, ICU MessageFormat, next-intl, BCP 47 locale negotiation, hreflang. Project: a tri-locale invoices list with localized dates, numbers, and currency.
- **Unit 19 — Testing**: Vitest 4, `expectTypeOf`, transaction-rollback integration tests against real Postgres, MSW v2, Playwright, React Testing Library, coverage thresholds. Project: a layered test suite for the Stripe webhook and Checkout money path (unit, integration, and E2E).
- **Unit 20 — Observability and Performance**: Sentry, Pino + AsyncLocalStorage, Vercel Drains, Vercel Analytics, PostHog (events, flags, replay), Core Web Vitals, `@next/bundle-analyzer`, Lighthouse CI. Project: wire Sentry, Pino, PostHog, and Vercel Analytics, then audit the app against Core Web Vitals.
- **Unit 21 — Git, CI, Deployment, and Migrations**: Git (rebase, bisect, cherry-pick), GitHub Actions, branch rulesets, Dependabot, Vercel, Fluid Compute, Vercel-Neon integration, OIDC federation, expand-migrate-contract migrations. Project: ship the app to production on Vercel, then run a live expand-migrate-contract schema change without an outage.
- **Unit 22 — Documentation and Code Review**: Diataxis, README discipline, AGENTS.md, ADRs (Nygard template), TSDoc, five-layer review stack, comment severity labels. Project: review a real PR through the five-layer review stack and write the accompanying ADR.
- **Unit 23 — AI with the Vercel AI SDK**: Vercel AI SDK v5, AI Gateway, multi-provider model handles (Claude/OpenAI/Gemini), tool calling, `streamText` chat, RAG, per-user token quotas. Project: an "ask-your-invoices" chat with tool calling and per-user daily token quotas.
