# The course

> A one-line compressed view of the [full Table of contents](../Table%20of%20contents.md). Each line names the tech, libraries, APIs, and architectural patterns introduced in that unit's teaching chapters and a brief description of the projects it includes.

- **Unit 1 — JavaScript and TypeScript (Chapters 001-009)**: TypeScript (generics, narrowing, discriminated unions, branded types), ES modules, Promises, async/await, AbortController, Temporal, JSON, mise + Node 24 LTS, VS Code config, native strip-types and `tsx`
- **Unit 2 — HTTP and the Browser Platform (Chapters 010-016)**: HTTP semantics, fetch, DOM, event handling, Server-Sent Events, Web Storage, cookies and `Set-Cookie` attributes, DevTools (Elements / Network / Console / Application)
- **Unit 3 — React, JSX, and Tailwind (Chapters 017-028)**: React 19, JSX, Tailwind v4, shadcn/ui, Radix UI, class-variance-authority, OKLCH design tokens, dark mode, container queries, responsive design, Next.js 16 project scaffolding from scratch, pnpm, TypeScript strict + project `tsconfig`, Biome, AGENTS.md. Project: a themed product surface built from scratch on a Next.js 16 + shadcn/ui scaffold.
- **Unit 4 — Next.js and the App Router (Chapters 029-035)**: App Router, Server/Client Components, Cache Components, Partial Prerendering, Suspense streaming, `proxy.ts`, `next/image`, `next/font`, `next/script`, metadata API. Project: a list-plus-detail surface using parallel routes with the modal-with-real-URL pattern, cloned from a `degit` starter (the first project in the course to start from a starter).
- **Unit 5 — Postgres and Drizzle (Chapters 036-041)**: Postgres on Neon, Drizzle ORM, Drizzle Relations v2, Drizzle Kit migrations, UUIDv7, indexes, transactions, `$inferSelect`/`$inferInsert`, `@t3-oss/env-nextjs` for build-time env validation. Project: the org-scoped invoicing data layer (schema, relations, migrations, seeds, env-driven `DATABASE_URL`).
- **Unit 6 — Forms, Validation, and Server Actions (Chapters 042-047)**: Zod 4, Server Actions, `useActionState`, native HTML forms, FormData, drizzle-zod, route handlers, `authedAction` wrapper, Result types. Project: a full CRUD surface implemented via Server Actions on the invoicing data layer.
- **Unit 7 — Transactional Email (Chapters 048-050)**: Resend, React Email, sender identity (SPF/DKIM/DMARC), deliverability. Project: the welcome email send path with a React Email template delivered through Resend.
- **Unit 8 — Authentication with Better Auth (Chapters 051-055)**: Better Auth, Drizzle adapter, email+password, session cookies, email verification, password reset, CSRF. Project: an end-to-end email+password auth flow with email verification.
- **Unit 9 — Organizations and RBAC (Chapters 056-059)**: Better Auth organizations plugin, RBAC, invitations, seat-handoff lifecycle, audit log, machine-caller API keys, multi-tenancy. Project: organizations, roles, and invitations wired end-to-end on the app.
- **Unit 10 — Lists, URL State, and Soft Delete (Chapters 060-062)**: URL state with `searchParams` + nuqs, soft delete / archive / restore, lifecycle columns, optimistic concurrency. Project: the production list view with URL-driven filters, sorting, pagination, and archive/restore.
- **Unit 11 — Webhooks and Stripe Billing (Chapters 063-065)**: Stripe (Checkout, Customer Portal, subscriptions, signing), webhook ingestion pattern, `processed_events` idempotency, plan entitlements. Project: a Stripe webhook pipeline that projects subscription events into a plan-entitlements row.
- **Unit 12 — Background Work and Object Storage (Chapters 066-069)**: Vercel `after()`, Vercel Cron, Trigger.dev v4 (queues, schedules, waitpoints, idempotency keys), Cloudflare R2, presigned URLs. Projects: a durable CSV export job on Trigger.dev, and a presigned-URL upload flow to R2.
- **Unit 13 — Notifications (Chapters 070-071)**: Notification dispatcher pattern, multi-channel send (email + inbox), notification preferences, dedup window. Project: a notification dispatcher fanning three events across email and in-app inbox channels.
- **Unit 14 — Cache and Rate Limiting (Chapters 072-075)**: Tag-driven cache invalidation (`updateTag` / `revalidateTag` / `revalidatePath`), Upstash Redis, `@upstash/ratelimit`, Vercel WAF, dual-keyed limiters. Projects: tag-driven cache invalidation on the invoices list, and Upstash rate limits on the auth surface.
- **Unit 15 — TanStack Query and Zustand (Chapters 076-079)**: TanStack Query v5 (`useQuery`, `useMutation`, `useInfiniteQuery`), Zustand v5 (`createStore`, slices), optimistic updates with rollback, polling. Projects: a polling, infinite-scrolling optimistic comment thread with TanStack Query, and a routed multi-step customer wizard with Zustand.
- **Unit 16 — Errors and Security (Chapters 080-082)**: Fail-closed error discipline, CSP with nonces, security headers, rate-limit coverage, audit log schema, GDPR retention, consent gating, pnpm 11+ supply-chain defaults. Project: a pre-launch audit pass that hardens error handling, headers, and the security baseline across the app.
- **Unit 17 — Time and Internationalization (Chapters 083-085)**: Temporal API, `timestamptz`/`date` codecs, IANA timezones, DST scheduling, `Intl.*` formatters, ICU MessageFormat, next-intl, BCP 47 locale negotiation, hreflang. Project: a tri-locale invoices list with localized dates, numbers, and currency.
- **Unit 18 — Testing (Chapters 086-091)**: Vitest 4, `expectTypeOf`, transaction-rollback integration tests against real Postgres, MSW v2, Playwright, React Testing Library, coverage thresholds. Project: a layered test suite for the Stripe webhook and Checkout money path (unit, integration, and E2E).
- **Unit 19 — Observability and Performance (Chapters 092-095)**: Sentry, Pino + AsyncLocalStorage, Vercel Drains, Vercel Analytics, PostHog (events, flags, replay), Core Web Vitals, `@next/bundle-analyzer`, Lighthouse CI. Project: wire Sentry, Pino, PostHog, and Vercel Analytics, then audit the app against Core Web Vitals.
- **Unit 20 — Git, CI, Deployment, and Migrations (Chapters 096-100)**: Git (rebase, bisect, cherry-pick), GitHub Actions, branch rulesets, Dependabot, Vercel, Fluid Compute, Vercel-Neon integration, OIDC federation, expand-migrate-contract migrations. Project: ship the app to production on Vercel, then run a live expand-migrate-contract schema change without an outage.
- **Unit 21 — Documentation and Code Review (Chapters 101-104)**: Diataxis, README discipline, AGENTS.md, ADRs (Nygard template), TSDoc, five-layer review stack, comment severity labels. Project: review a real PR through the five-layer review stack and write the accompanying ADR.
- **Unit 22 — AI with the Vercel AI SDK (Chapters 105-108)**: Vercel AI SDK v5, AI Gateway, multi-provider model handles (Claude/OpenAI/Gemini), tool calling, `streamText` chat, RAG, per-user token quotas. Project: an "ask-your-invoices" chat with tool calling and per-user daily token quotas.
