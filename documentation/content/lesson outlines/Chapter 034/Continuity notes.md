## Lesson 3 — Edge redirects and rewrites

**Cut** — `headers()` config *contents* carried as a commented placeholder only (→ ch.081); async `redirects()` data sources (CMS/manifest-generated rules) dropped entirely (outline allowed one sentence — not even that); `statusCode` escape hatch omitted (one-clause "use `permanent`, never `301`/`302`" covers the method-rewrite footgun); i18n/`locale` redirect forms omitted (→ ch.084); `basePath` omitted.

**Debts** — security-header *contents* (`headers()`: CSP, HSTS) → **ch.081** (whole-config worked example carries the commented `// async headers()` signpost); validation/authz (vs. presence) → `proxy.ts` (ch.033) then route; i18n routing / `hreflang` → ch.084. Reuses ch.033 l3 (redirect-vs-rewrite, 307/308, link equity, loops), ch.029 l4 (`redirect()`/`permanentRedirect()`), ch.034 l1 (config read once at startup → dev-restart gotcha, recapped here as promised).

**Terminology** — `Term`-defined here: **link equity** (ranking value a 308 forwards, a 307 doesn't), **path-to-regexp** (the `source` pattern library — same `:param`/`*` grammar as Express), **reverse proxy** (external rewrite streams upstream through your deployment). Spine one-liner (memorize): *"static-and-known → config; request-conditional → proxy; after-an-action → `redirect()`"* — and the routing question *"does this rule depend on who's asking?"* asked first. Subtlety: presence-check (cookie exists?) stays in config; validation-check (is the session valid?) moves to proxy.

**Patterns and best practices** — Request-blind URL rules belong in `next.config.ts` (edge, free), never `proxy.ts` (pays an invocation per matched request). Every `source` starts with `/` (leading slash anchors the pattern — its absence is the canonical infinite-redirect-loop cause). `:slug`=one segment, `:path*`=whole sub-tree (the rebrand workhorse); pick `:path*` for section renames. `has`/`missing` for UX nudges only, never security decisions. External-rewrite matchers stay tight to the real prefix (`/docs/:path*`), **never bare root `/:path*`** (would proxy `/_next/*` assets and API routes). `proxy.ts` rewrite wins over a config rewrite when both match — one home per rule, never split. `trailingSlash` decided once at project start and locked (default `false` for SaaS); flipping post-launch is an irreversible redirect tax on every backlink. Config conventions inherited from l1: `import type { NextConfig }`, single quotes, 2-space.

**Misc.** — Chapter-wide anchor scenario is the **`/account` → `/settings` rebrand** (request-blind, permanent — the self-evident config rule); reuse for any later l3-adjacent example. The chapter's **blue continuity-anchor `next.config.ts`** now carries `cacheComponents`/`typedRoutes`/`trailingSlash: false` + `redirects()` (rebrand 308 + `missing`-cookie 307) + external `rewrites()` (`/docs` → `https://docs.acme-marketing.com`) + commented `headers()` placeholder — under 30 lines, the artifact later config lessons extend. **Terminology** — `Term`-defined here: **FOUT** (fallback shows then swaps), **FOIT** (text hidden until font loads — the worse failure), **woff2**. CLS reused from l2 (not re-defined). Spine one-liner: *"`next/font` turns a render-blocking, privacy-leaking `<link>` into a self-hosted, zero-CLS, build-time asset — and the variable bridge turns each font into a Tailwind token."* Key distinction: the `.variable` className (sets a CSS custom property) vs the bare `.className` (sets `font-family` directly) — mixing them is the documented footgun.

**Patterns and best practices** — Five review reflexes: (1) declare `subsets` on every Google font (omission = build **warning** not error, where byte-savings + preload live); (2) variable fonts omit `weight`, static fonts list only rendered weights/styles — prefer variable; (3) wire fonts through `variable` + `@theme` in `globals.css`, **never `tailwind.config.ts`** (a font is a design token like a color; v4 is CSS-first); (4) call the loader at **module scope, once**, never inside a component (re-renders re-run the loader); (5) `next/font` needs **zero `next.config.ts`** (contrast with images/redirects). Also: never load a font in a Client Component; only pass `className`/`variable`/`style` across the server/client boundary, never the font object; loading scope matches render scope (body font in root layout, marketing face in `(marketing)` layout only); `app/fonts.ts` exports one instance per font, imported everywhere. **There is no `tailwind.config.ts` in this course** — the chapter outline's `fontFamily.sans: ['var(--font-sans)']` config shape is the corrected legacy-v3 trap.

**Misc.** — Semantic font-variable naming (`--font-sans`/`--font-mono`/`--font-display`), **not** `--font-geist` (keeps tokens swappable). Local brand face is `acme-display.woff2` at `app/fonts/` (coherent with `@acme/*` vendor convention). **Terminology** — `Term`-defined here: **main thread**, **hydration** (refresh), **Web Worker** (background thread, no DOM access), **Partytown**, **GDPR**, **ePrivacy**. Spine one-liner (the durable senior question, two outfits): *"what breaks if this script loads five seconds late, or disappeared tomorrow?"* — drives strategy choice (→ `lazyOnload`/delete) and integration choice (→ typed SDK over a tag). ## Lesson 8 — generateStaticParams for static catalogs
