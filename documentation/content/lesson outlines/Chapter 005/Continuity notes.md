- **Terminology** — `<Term>` tooltips installed for **generic** ("A function or type that accepts a type as a parameter. The call site supplies it explicitly (`identity<number>(42)`) or by inference (`identity(42)`)."), **type parameter** ("The `<T>` declaration. Names a type the caller will fill in. Used in parameter types, return types, and the function body's type annotations."), **default type parameter** ("A type parameter declared with `= DefaultType`. If the caller omits the position, the default fills in. Like value-parameter defaults, with one tail rule: a defaulted parameter cannot precede a non-defaulted one."), **constraint** ("An `extends` clause on a type parameter (`<T extends ...>`) that requires the caller's type to be assignable to the right-hand side.
