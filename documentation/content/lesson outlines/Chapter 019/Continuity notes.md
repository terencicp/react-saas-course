# Chapter 019 — The cascade, inheritance, and design tokens

## Lesson 1 — How the browser picks a winning rule

**Taught** — The four-gate cascade algorithm as a short-circuiting waterfall (origin & importance → cascade layer → specificity → source order), Tailwind v4's four layers (`theme`/`base`/`components`/`utilities`), the unlayered-beats-everything trap, specificity tuples and `:where()`/`:is()`, why `!important` is a smell with two carve-outs, source order resolved by `cn()`, and the DevTools Computed-panel trace move.

**Cut** — Chapter-outline brainstormed a `Matching` exercise and a `Screenshot`/`CodeReview` variant; lesson shipped a mock-free real-component approach instead (custom `CascadeWaterfall`/`LayerStack`/`SpecificityTuple`/`ComputedPanelTrace` lesson components). No load-bearing scope cut.

**Debts** — Opening heading bug's "no large default" resolution is deferred to lesson 3 (Preflight); inline `style={{}}` for *dynamic* values (CSS custom props at runtime) deferred to lesson 4; `cn()`/`tailwind-merge` internals belong to Ch018 L3 (cross-ref only); inheritance (what flows down when no rule applies) is lesson 2.

**Terminology** — "four-gate waterfall" / "gate 1–4" naming for the cascade steps; "gate 2 / the layer gate decides almost everything on Tailwind v4"; "layer, not bang" senior reflex; specificity written as 4-tuple `(inline, ID, class/attr/pseudo-class, element/pseudo-element)`; "important inversion" (earlier layer wins for `!important`); the "three-click move" (Computed → expand property → read winning rule + its layer).

**Patterns and best practices** — Every custom CSS rule names its layer: `@layer base` for global element baselines (resets, `h2`/`code`/`kbd`/link defaults), `@layer components` only when no utility composition works; never leave CSS unlayered; never reach for `!important` or higher specificity to win a self-authored conflict (move to correct layer instead); never use inline `style` to win a static conflict; run conflicting utilities through `cn()`. Project `globals.css` mirrors Ch018 shape (`@import "tailwindcss"`, `@custom-variant dark (&:is(.dark *))`).

**Misc.** — Lesson handles the deliberate `:is(.dark *)` (shipped course code) vs `:where(.dark, .dark *)` (Tailwind docs) discrepancy explicitly: teaches `:where()` as the specificity-zero concept, notes the shipped `:is()` form scores `.dark`'s single-class specificity and is fine because the toggle sits on `<html>`. Do not "correct" the shipped form in later lessons. Lesson components live at `src/components/lessons/019/1/`.

## Lesson 2 — What flows down the DOM tree

**Taught** — CSS inheritance as a per-property spec flag (inherited yes/no); the substitution mechanic (no declared value → inheriting copies parent's *computed* value, non-inheriting uses spec *initial* value); the text-vs-box heuristic for predicting families; inheritance flows the **DOM tree, not the JSX tree**; the `inherit`/`initial`/`unset`/`revert`/`revert-layer` keywords at recognition level; form controls as inheritance-rebels (UA stylesheet sets `font`/`color` directly, defeating inheritance) with Preflight's `font: inherit; color: inherit` named as the fix; `currentColor` for SVG icons and borders; four look-alikes (`opacity` compositing, `visibility`/`display`, transparent-background passthrough, `em` compounding); and the closing placement rule.

**Cut** — Outline brainstormed a `TabbedContent`/`Screenshot` DevTools figure and `Matching`; shipped custom lesson components instead (`InheritanceDrip`/`InheritsVsNot`/`InheritedFromPanel`/`LooksVsActuallyInherited`) plus `ParamPlayground`, `Buckets`, `MultipleChoice`, `TrueFalse`, `ReactCoding`. No load-bearing scope cut.

**Debts** — Full Preflight reset surface + `@layer base` location deferred to L3 (only the form-control `font: inherit` rule named here); border `border: 0 solid currentColor` reset attributed to Preflight, full story L3; custom-property inheritance named as *why* `.dark` on `<html>` reaches everything but the three-tier token model deferred to L4; `em`-vs-`rem` reflex named, type scale deferred to Ch021 L1; OKLCH/color deferred to Ch021 L2; box model + `box-sizing` to Ch020 L1; `display`/`visibility`/`aria-hidden` hiding decision tree to Ch020 L2; shadcn `<Button>`/`<Input>` and SVG icon component patterns to Ch027 (recognition cross-ref only). Repays L1 debt: explains *why* bare `<button>` ignores body typography.

**Terminology** — "inheritance is a per-property flag" (not a global switch); "text-ish inherits, box-ish doesn't" heuristic; "computed value of the parent" (load-bearing — sets up `em` compounding); "inheritance rebels" for form controls; "an inherited value is the weakest source in the cascade / the absence of a declaration" (reused from L1, beats any real declaration); "typography up, box-model down" placement rule; "text styles flow from `<body>`, tokens flow from `:root`"; `currentColor` = computed value of `color` on the element.

**Patterns and best practices** — Put typography + color utilities on `<body>` / high ancestor (they inherit); put box-model + layout utilities per element (they don't); tokens on `:root`/`.dark`. Don't hand-set fonts on form controls or fight Preflight — it re-inherits them. Don't pass styles via props/wrapper components expecting inheritance (it walks DOM, not JSX). Prefer `rem` over `em` (Tailwind `text-*` scale emits `rem`, no compounding). Use `fill-current`/`stroke-current` or rely on Lucide's `currentColor` default + color the parent. Raw `inherit`/`initial` in hand-written CSS is a yellow flag (organize with layers instead). Leaf components carry zero repeated typography classes.

**Misc.** — Examples deliberately show *raw* `font: inherit`/`color: inherit` CSS (Preflight's rule) and the bare-`<button>`-without-Preflight state as pedagogical, not project code — do not "correct" to utility form. Illustrative `text-red-500`/`text-indigo-600` used as throwaway demo values (not semantic tokens) deliberately. Lesson components live at `src/components/lessons/019/2/`.

## Lesson 3 — Preflight, the deliberately blank canvas

**Taught** — Preflight as Tailwind v4's built-in reset loaded by `@import "tailwindcss"` into `@layer base` (so utilities in `@layer utilities` always win the layer gate, never need `!important`); the `@import` expansion to four layers; what it strips, named not exhaustively (margins/padding everywhere, `box-sizing: border-box`, `border: 0 solid`, headings `font-size`/`font-weight: inherit`, lists `list-style: none`, form controls `font/color: inherit`, block+constrained media, link underline/color); the diagnosis skill (reset-doing-its-job vs real bug); reading Preflight in the DevTools Styles panel `@layer base` band; the two carve-outs (`prose` plugin for un-owned content, scoped `@layer base` + `revert` + `:where()` override for third-party widgets); the three rejected fixes (strip globally, global `h1 {}`/`@apply`, `!important`).

**Cut** — `cursor: pointer` on buttons (chapter outline listed it; v4 removed it — taught explicitly as *not* a Preflight rule, you add `cursor-pointer` or shadcn does); the full `@tailwindcss/typography` API and prose theming depth; the omit-Preflight mechanism (recognition only). No load-bearing cut.

**Debts** — `prose` themeability via `@theme` tokens (`--prose-body`/`--prose-headings`) named only; empty-state `prose` cash-in deferred to Ch027 L5; box model + `box-sizing` semantics to Ch020 L1; `gap`-over-sibling-margins reflex named, deferred to Ch020 (box-model chapter); shadcn `<Button>`/`<Input>` as the real surface built on Preflight to Ch027 + Ch017 L5 (recognition); type scale to Ch021 L1; placeholder `color-mix`/OKLCH to Ch021 L2; `@tailwindcss/forms` named as *not shipped* (shadcn doesn't need it). Repays L1 debt (bare `<h1>` has no large default to fight) and L2 debts (form-control `font: inherit`, `border: 0 solid` load-bearing reset).

**Terminology** — "the reset is doing its job; the utility is my job" (the lesson's recognition mantra, reused from L1/L2); "blank canvas" / "broke my HTML → blank canvas" flip; "Preflight is structurally beneath everything you write"; "every pixel of visual weight comes from a utility you wrote"; `border: 0 solid` is "load-bearing" — `border` utility sets *width* only and renders solely because Preflight pre-set style to `solid` (no Preflight → `border` paints nothing); the `prose`/`prose-invert`/`prose-sm`/`prose-lg`/`max-w-prose`/`max-w-none` family.

**Patterns and best practices** — Never strip Preflight globally; never `@apply h1 {}` or write global element rules to "restore" defaults (put utilities on owned markup; use `prose` for un-owned content); never `!important` against Preflight (the layer gate already wins). Bring defaults back only via the two scoped carve-outs: `prose` (install `@plugin "@tailwindcss/typography"`, wrap in `prose dark:prose-invert max-w-prose`) for Markdown/CMS blobs, or a `@layer base { .widget :where(...) { ...: revert } }` scoped override for third-party widgets. Components are built on the cleared canvas, not against it.

**Misc.** — Continues the convention of showing *raw* literal Preflight CSS (margins/box-sizing/headings/lists/form-control rules) as pedagogical, not project code — do not "correct" to utility form. `text-2xl font-bold`/`space-y-*` used as throwaway demo utilities. Lesson components live at `src/components/lessons/019/3/` (`PreflightStrip` for the `DiagramSequence`, `PreflightInBaseLayer` for the DevTools Styles-panel mock). Shipped one `VideoCallout` (Tailwind Labs prose walkthrough, `GEYkwfYytAM`).

## Lesson 4 — Custom properties and the three-tier token model

**Taught** — Custom property (`--*`) as a **binding, not a constant**: inherits down the DOM tree and re-resolves live at use time against the property's computed value; every `@theme` token is a `:root` custom property and every utility (`bg-card`) compiles to a `var()` read; subtree overrides (`:root` default → `.dark`/`.marketing-hero`/`[data-org]` re-declaration) re-theme a whole subtree with components unchanged; the JS DOM API (`style.setProperty`/`getComputedStyle().getPropertyValue`/`style.removeProperty`) as a **visual channel that does not re-render React**; React `style={{ '--x': v }}` + arbitrary-value utility (`p-[var(--x)]`) for dynamic per-instance tokens; the three-tier model (primitive = value, semantic = role components read, component = rare exception); purpose-not-appearance naming; `@property` for typed, animatable custom properties.

**Cut** — Optional second `ParamPlayground` for the resolution chain (shipped the `TokenResolutionChain` figure as primary instead); optional `Matching`/`Dropdowns` bad-name→good-name exercise in the naming section (prose + earlier `Buckets` carry it); `:root`-vs-`html` specificity nuance folded into one parenthetical. No load-bearing scope cut.

**Debts** — Repays L1 (dynamic inline `style={{}}` for per-instance values, the legitimate inverse of L1's "inline `style` is a smell for static conflicts") and L2 (custom-property inheritance → why `.dark` reaches everything). Defers: color-space/OKLCH channel meaning to Ch021 L2 (uses OKLCH literals without explaining them); CSS animation/transition keyframe syntax to Ch021 L5 (`@property` names only the enabling primitive); shadcn's exact semantic-token list to Ch027 L1 (named "the semantic tier you'll meet," not enumerated); `[data-org]` multi-tenant theming as a real feature to Unit 9 (orgs) — named as a use case only; FOUC pre-paint inline-`<script>` cure cross-refs Ch018 L6 (`next-themes`), recognition only.

**Terminology** — "**binding, not a constant**" (the load-bearing reframe); "visual channel vs state channel" (JS/`setProperty` paints pixels, React state holds the truth — a color-picker writes the property on `onInput` for the drag, commits to state on `onChange`); "override down the tree" / "declare the default on `:root`, override on a descendant selector"; "arbitrary-value escape hatch" (`p-[var(--name)]`); three tiers named **primitive / semantic / component**; "the token name is an API" (renaming = breaking change); "a token animates but snaps → it's an unregistered custom property" (the `@property` recognition).

**Patterns and best practices** — Components reference the **semantic** tier, never primitives (`bg-primary`, never `bg-blue-600` — direct primitive use in a component is a smell); semantic token points at a primitive via chained `var()` (`--color-primary: var(--blue-600)`) so a rebrand is one pointer change; pair every surface with a `--color-{role}-foreground` for guaranteed-contrast text; express states with opacity modifiers (`bg-primary/80`) not new `-hover` tokens; name tokens for purpose not appearance (`--color-destructive`, not `--color-red`) across every namespace (`--spacing-{role}`/`--radius-{role}`/`--shadow-{role}`); component tier is a deliberate rare exception; for live theme writes from JS, write the property for instant feedback **and** commit to React state/persistence; runtime tokens from server data need a pre-paint inline-`<script>` write to avoid FOUC. Arbitrary-value `var()` utilities are the exception, semantic tokens the rule.

**Misc.** — Deliberate divergences from convention, shown as the *taught* pattern (not the smell): arbitrary-value utilities (`p-[var(--x)]`), raw inline `style={{ '--x': ... }}`, bare primitive utilities (`bg-blue-600`) only as the anti-example, and raw `--color-primary: var(--blue-600)` CSS as tier-chain illustration. Keep shipped `globals.css` verbatim (plain `@theme`, not `@theme inline`); `@theme inline` appears only as the named one-line watch-out for why it would *sever* runtime overrides (it bakes the literal value into the utility; a `var()`-pointing token like shadcn's `--color-card: var(--card)` stays live). One new lesson component: `TokenResolutionChain` at `src/components/lessons/019/4/`.
