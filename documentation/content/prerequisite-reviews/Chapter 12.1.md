# Chapter 12.1 prerequisites review

## Missing prerequisites

- <Lesson 12.1.1> — Stripe Node SDK setup (installation, `lib/stripe.ts` singleton, `STRIPE_SECRET_KEY` env). Quote: "The Stripe Node SDK exposes `stripe.webhooks.constructEvent(body, signature, secret)` which encapsulates the timestamp parse, HMAC compute, constant-time compare, and tolerance check." The worked handler also calls `constructEvent` directly and references `STRIPE_WEBHOOK_SECRET` as an env entry. The SDK singleton and secret-key discipline are first introduced in Lesson 12.2.1 ("The Stripe Node SDK — what the course uses"), which comes after this chapter. Suggested source: new lesson in Unit 12 before 12.1 (e.g., 12.0 or an opening section of 12.1.1 that installs the SDK) or move the SDK setup content from 12.2.1 to a pre-12.1 stub. Severity: high.

- <Lesson 12.1.3> — `plan_entitlements` table schema. Quote: "update plan_entitlements set status = ${new}, last_event_at = ${event.created} where org_id = ${id} and (last_event_at is null or last_event_at < ${event.created})". The worked example executes a concrete SQL UPDATE against this table, but the table's schema (columns, constraints, purpose) is only introduced in Lesson 12.2.4, which is after this chapter. The lesson does list "The `plan_entitlements` derived view and the full Stripe state model — Chapter 12.2" in its "does not cover" section, but a student following the lesson code cannot run the worked example without the schema. Suggested source: 12.2.4 or a brief inline schema sketch added to 12.1.3's worked-example bullet. Severity: low.
