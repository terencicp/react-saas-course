# Chapter 4.7 prerequisites review

## Missing prerequisites

- <Lesson 4.7.3> — `useState` basic call syntax used as if known. Quote: "`useState(expensiveCompute())` runs `expensiveCompute()` on every render even though only the first call's result is used." The lesson builds its initializer-trap example on the assumption that students can read `useState(expr)` and know what it does. The `useState` API surface is explicitly deferred to 4.8.1 ("What this lesson does not cover: `useState` API surface, lazy initial state, and derived state — Chapter 4.8"). Suggested source: introduce the one-line destructuring form (`const [value, setValue] = useState(initial)`) either at the end of 4.6.1 or as a brief bridge at the start of 4.7.1. Severity: high.

- <Lesson 4.7.4> — `useState` setter and destructuring pattern used as the central teaching example before the API is taught. Quote: "`function handleClick() { setCount(count + 1); setCount(count + 1); setCount(count + 1); }` increments by 1, not 3, when `count` starts at 0." The full lesson ("State is a snapshot") is built on `const [count, setCount] = useState(0)` as assumed prior knowledge, including the updater form `setCount(c => c + 1)` and `const [resetKey, setResetKey] = useState(0)` in the reset-key pattern. None of this syntax has been introduced before 4.7. The `useState` API surface is explicitly deferred to 4.8.1. Suggested source: introduce the basic `useState` destructuring form in 4.6.1 or as a short bridge lesson before 4.7, or reorder so 4.8.1 precedes 4.7.4. Severity: high.

- <Lesson 4.7.6> — `createRoot` used as an assumed known concept to explain React's event delegation point. Quote: "React attaches one listener per event type at the application's root container (the element passed to `createRoot`) rather than at `document`." The concept of React's root container and the `createRoot` call is never introduced in any prior chapter (not in 4.1 through 4.6, which cover JSX, Tailwind, CSS, and component composition). Students working through a Next.js course never call `createRoot` themselves, but understanding *where* React's event root sits requires knowing that `createRoot` mounts the React tree into a specific DOM node. Suggested source: add a one-paragraph explanation of the React root mount point to 4.1.2 ("The Next.js root layout owns the document shell"), which already discusses the `app/layout.tsx` shell — or add a brief note to 4.7.6 itself before invoking the concept. Severity: low.
