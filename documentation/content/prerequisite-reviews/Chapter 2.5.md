# Chapter 2.5 prerequisites review

## Missing prerequisites

- Lesson 2.5.2 — `AbortController`. Quote: "states are `{ kind: 'idle' } | { kind: 'uploading'; progress: number; controller: AbortController } | { kind: 'done'; url: string } | { kind: 'error'; error: Error }`; the student should leave able to read this shape without commentary." `AbortController` is used as a per-state field type in the canonical upload machine, presented as readable without explanation. It is not introduced until lesson 2.7.4 ("Cancellation with `AbortController` and `AbortSignal`"), which comes after this chapter. Suggested source: 2.7.4, or add a one-line definition note in 2.5.2 acknowledging the forward reference. Severity: medium.

- Lesson 2.5.7 — `ZodType` and `z.infer<Input>` as understood generic constraint concepts. Quote: "Signature shape: `function safeAction<Input extends ZodType, Output>(schema: Input, handler: (input: z.infer<Input>) => Promise<Output>): (input: unknown) => Promise<Result<Output, AppError>>`." The lesson requires the student to read and understand `ZodType` as the base type all Zod schemas satisfy and `z.infer<>` as the type-extraction operator. Neither is taught before Chapter 2.5 — 1.4.5 shows Zod schema values (`z.url()`, `z.string().min(1)`) but never the `ZodType` TypeScript type or `z.infer`. Full Zod type depth lands in 7.1.4. Suggested source: add a one-line definitional note in 2.5.7 ("`ZodType` is the base TypeScript type that every Zod schema satisfies; `z.infer<S>` extracts the output type of schema `S`") or move the `safeAction` signature example to a post-Unit-7 lesson. Severity: medium.
