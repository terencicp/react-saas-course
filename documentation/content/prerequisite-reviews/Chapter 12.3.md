# Chapter 12.3 prerequisites review

## Missing prerequisites

- Lesson 12.3.3 — structured logging API (Pino). Quote: "Wire the structured logger from Chapter 20.1 (provided in the starter as `lib/logger.ts`): every verification path logs `{ eventId, eventType, disposition }` where disposition is `'verified'`, `'invalid_signature'`, or `'missing_header'`." The lesson treats structured logging as something the student already knows how to use, names the concept by its Chapter 20.1 home, and instructs the student to wire it against a real production seam ("The structured log is what 2am debugging will need"). Chapter 20.1.2 (Structured logs with correlation IDs — Pino, fixed JSON key set, child loggers) is in Unit 20, after Unit 12. The starter provides `lib/logger.ts` and the lesson prescribes exact call shapes, so the student can follow mechanically, but the concept of why structured key-value logging differs from `console.log` is not established. Suggested source: introduce a minimal structured-logging primer (what `pino` is, the key-value call shape, why machine-parseable JSON over string concatenation) either in Chapter 12.1 where the same gap exists, or add a dedicated lesson before Chapter 12.1. Severity: medium.
