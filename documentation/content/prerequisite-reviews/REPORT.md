## High-severity findings (12)

Concepts used as load-bearing knowledge with no prior teaching anywhere in the curriculum.

- **[Chapter 2.7](Chapter%202.7.md)** — `try`/`catch` syntax and mechanics. The `return await` rule in 2.7.3 and the `catch (err) { if (err instanceof Error && err.name === 'AbortError') return; throw err }` shape in 2.7.4 require `try`/`catch`, but 2.2.4 explicitly defers it to 2.8.1.
- **[Chapter 2.7](Chapter%202.7.md)** — same `try`/`catch` gap in the 2.7.4 abort handler.
- **[Chapter 3.7](Chapter%203.7.md)** — `Uint8Array` construction and the typed-array model. 3.7.1's HMAC flow uses `new Uint8Array(32)` and the `ArrayBuffer` relationship as known; 3.6.2 names `Uint8Array` only as a chunk type, never as a constructable buffer.
- **[Chapter 3.7](Chapter%203.7.md)** — `TextEncoder` / `TextDecoder` as the string↔bytes primitive. Used as a known API in 3.7.1; 3.6.2 mentions them inside a streaming snippet but never introduces their call shape.
- **[Chapter 4.7](Chapter%204.7.md)** — `useState` call syntax in the 4.7.3 initializer-trap example. API surface explicitly deferred to 4.8.1, but 4.7.3 reads `useState(expr)` as known.
- **[Chapter 4.7](Chapter%204.7.md)** — `useState` destructuring + setter as the central teaching example of 4.7.4 ("State is a snapshot"). Whole lesson built on `const [count, setCount] = useState(0)` and the updater form, none of which is taught before 4.7.
- **[Chapter 10.3](Chapter%2010.3.md)** — `crypto.subtle.digest` SHA-256 at implementation depth. 3.7.1 names `digest` only at recognition level; 10.3.1 uses the full pipeline (`crypto.subtle.digest('SHA-256', buf)` → `ArrayBuffer` → bytes → store) as known.
- **[Chapter 12.1](Chapter%2012.1.md)** — Stripe Node SDK setup. 12.1.1's worked webhook handler calls `stripe.webhooks.constructEvent(...)` and the lesson references `STRIPE_WEBHOOK_SECRET`, but the SDK singleton and secret-key discipline are first taught in 12.2.1 — *after* 12.1.
- **[Chapter 13.1](Chapter%2013.1.md)** — Vercel Fluid Compute time caps (`maxDuration`, 14-min Pro, 1-min Hobby). Used as established threshold facts throughout 13.1.1–3, but Fluid Compute isn't taught until 21.3.3 — eight units later.
- **[Chapter 13.4](Chapter%2013.4.md)** — `XMLHttpRequest` call surface. 13.4.4 has the student write `xhr.open`, `xhr.upload.onprogress`, `xhr.setRequestHeader`, `xhr.send` from scratch for R2 upload progress. 3.6.1 names XHR "recognition only" and defers teaching to *this* chapter.
- **[Chapter 16.3](Chapter%2016.3.md)** — Architectural Principle #6 numbering collision. 5.2.3 binds #6 to "prefer explicit over magic"; 16.3.1 rebinds the same number to a different rule ("per-feature, never global ambient"). One number, two principles.
- **[Chapter 17.2](Chapter%2017.2.md)** — Husky / pre-commit hook setup. 17.2.6 instructs Gitleaks-in-pre-commit and cites "Husky from Unit 21.2.4" — but Chapter 1.3 defers Husky to Unit 21 and Chapter 21.2.3 reflects it back to Unit 1. Husky never gets taught anywhere.

## Themes across the course

Repeated patterns surface curriculum-level structural choices rather than one-off slips. Each is worth a deliberate decision about reordering vs. forward-reference framing.

1. **React hooks used before they're owned (`useState`/`useEffect`).** 4.2.6, 4.6.5, 4.7.3, 4.7.4, 4.8.2, 4.9.1 all use hooks as known before 4.8 (state) and 4.9 (effects) formally introduce them. 4.7.4's entire "State is a snapshot" lesson rides on it. Either reorder 4.8/4.9 to precede 4.7, or scope 4.7's examples to non-hook DOM equivalents.

2. **TypeScript object-type syntax in Unit 2 before 2.4 owns it.** 2.2.2, 2.2.6, 2.2.4 use inline `{ name: string; admin?: boolean }` annotations, `unknown`, and `assertNever`/`never` as canonical patterns the student must read — but `?`-optional fields, `unknown`, and the `never` bottom type are introduced in 2.4.1/2.4.2/2.4.6. Cluster of medium-severity gaps; would resolve if 2.4 came before 2.2, or if 2.2 confined examples to inferred types.

3. **`try`/`catch` taught after async (Unit 2.8) but assumed throughout Unit 2.7.** The `return await` rule and abort handler patterns can't be read without `try`/`catch`. Single fix: move 2.8.1's `try`/`catch` introduction to early Unit 2.

4. **Byte-level Web primitives (`Uint8Array`, `TextEncoder`/`TextDecoder`, base64url).** 3.6 streams chunks past them without teaching, then 3.7.1 builds HMAC on the full surface. Add a short primer at the top of 3.6.2 (or a new short lesson 3.6.x) covering `Uint8Array` construction, `TextEncoder`/`TextDecoder` shape, and base64url encoding.

5. **App Router primitives referenced before Unit 5.** Server Components / Server Actions / Route Handlers / `NextResponse` show up in 2.9.1, 3.3.3, 3.4.1 as labeled examples or worked code. Suggested fix: either a one-paragraph App Router orientation in Unit 3.1, or rewrite the examples to be runtime-neutral until Unit 5 owns them.

6. **Architectural Principles cited before they're named.** Principle #3 invoked in 1.4.1, 5.7.2, 7.1.3 — formally introduced in 7.2.4. Principle #6 used in 5.2.3 *and* re-bound to a different rule in 16.3.1 (high-severity numbering collision). A single-source-of-truth list of principles with their owning lesson IDs (and a forward-reference convention before that lesson) would close the whole class.

7. **Core Web Vitals (LCP, INP, CLS, p75 thresholds) used before 20.3.1 defines them.** 4.4.5, 5.6.2, 20.2.1 all assume the acronyms. Either define them once at first use (4.4.5) or reorder 20.3.1 before 20.2.

8. **CI / GitHub Actions wiring requested before 21.2 owns the primitives.** 17.2.6 (Husky/pre-commit), 20.3.4 (analyzer artifact upload), 20.3.5 (LHCI in CI), 21.4.3 (`neonctl` in CI) all instruct workflow authoring before GHA workflow/job/step model is taught. Move CI wiring of these tools into Unit 21.2, or add a short pre-21.2 GHA primer.

9. **Vendor SDKs anticipated before the unit that installs them.** Stripe SDK before 12.2.1 (12.1.1, high); Trigger.dev & R2 SDKs cited in 12.2.7 before Unit 13; PostHog SDK methods called in 17.2.5 before 20.2.3; `neonctl` in 21.4.3 before any introduction. Common root: cross-cutting chapters reference SDKs already; either install the SDK in a tiny pre-chapter or restrict examples to the SDK already in hand.

10. **Drizzle `logger: true` is the recommended N+1 diagnostic in 6.4.2/6.4.3 but isn't wired until 6.5/6.2.2.** Two medium findings in one chapter; trivial to fix by moving the one-line client-config setup earlier.

11. **Structured logging / Pino assumed before 20.1.2.** 12.3.3 prescribes exact Pino call shapes; 17.2 references the same convention. A 50-word primer in Unit 12.1 (or earlier) would unblock both.

12. **`useSearchParams` / `usePathname` used before 5.5.5.** 4.10.1, 4.11.4 examples assume these App Router hooks. Low-severity individually, but cluster suggests an earlier brief naming would help.

## How to use this report

Each per-chapter file follows the agent's required structure (`## Missing prerequisites`, lesson ID, quoted snippet, severity). The 12 high-severity items are the load-bearing ones — they break comprehension, not just slow it. The themes section identifies where the same fix unblocks several findings at once. The 51 clean chapters provide implicit confirmation that the dominant authoring discipline is sound; the gaps are concentrated, not pervasive.
