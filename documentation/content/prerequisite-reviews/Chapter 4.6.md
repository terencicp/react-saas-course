# Chapter 4.6 prerequisites review

## Missing prerequisites

- **Lesson 4.6.5** — `useState` and `useEffect` used as known APIs. Quote: "a `useEffect` mount flag (`const [mounted, setMounted] = useState(false); useEffect(() => setMounted(true), [])`)". Neither hook is formally introduced before this chapter; 4.8.1 owns `useState` and 4.9.2 owns `useEffect`. The informal pass in 4.2.6 ("a `useState`-tracked `mounted` flag flipped in `useEffect`") presents the pattern without teaching the APIs. The chapter's own scope note in 4.6.1 lists "State and hooks — Chapters 4.8 and 4.9" as not covered. Suggested source: add a one-line forward reference in 4.6.5 to 4.8.1/4.9.2 alongside the alternative `typeof window !== 'undefined'` form. Severity: low.

- **Lesson 4.6.4** — `IntersectionObserver` used as if already known in a ref-callback code example. Quote: "the form: `ref={(node) => { const observer = new IntersectionObserver(...); observer.observe(node); return () => observer.disconnect(); }}`". Chapter 3.5 explicitly defers `IntersectionObserver` ("covered as they earn their weight in later chapters — lazy-loading, layout, virtualization"); no chapter between 3.5 and 4.6.4 introduces it. It serves as the motivating example for the React 19 ref-callback cleanup return, so the reader must understand what it does to follow the lesson. Suggested source: add a one-sentence gloss at first use ("an `IntersectionObserver` watches when a node enters the viewport"), or defer the example to a chapter after the observer APIs are introduced (4.9 or 4.10). Severity: low.
